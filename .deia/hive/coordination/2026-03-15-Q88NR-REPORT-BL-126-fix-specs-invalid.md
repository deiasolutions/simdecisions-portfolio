## Root Cause Analysis

### The Queue Runner Bug

**Current behavior (BROKEN):**
1. Queue runner processes spec X
2. After bees complete, queue runner runs full test suite
3. Queue runner sees ALL test failures in suite
4. Queue runner ASSUMES all failures are regressions from spec X
5. Queue runner generates fix spec listing all failures

**Why this fails:**
- The test suite has pre-existing failures from other commits
- Not all failures are caused by spec X
- No way to distinguish NEW failures from PRE-EXISTING failures

**Correct behavior (NEEDED):**
1. Queue runner snapshots test results BEFORE processing spec X
2. Queue runner processes spec X
3. Queue runner runs full test suite after bees complete
4. Queue runner DIFFS test results (before vs after)
5. Queue runner ONLY flags NEW failures as regressions
6. If no new failures, mark spec X as COMPLETE
