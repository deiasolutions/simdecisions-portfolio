## Objective

Diagnose and fix why PHASE-IR mutations generated by the LLM in the terminal/chat pane are not being applied to the canvas in the canvas2 EGG.

---

## Context

The user reports that in the canvas2 EGG:
1. User types in IR terminal (paneId `canvas-ir`, routeTarget `ir`)
2. LLM responds with PHASE-IR JSON mutations
3. **Those mutations never reach the FlowDesigner to create/update nodes on the canvas**

The expected IR pipeline is:
1. User types in terminal → LLM responds with mixed text + JSON
2. `terminalResponseRouter.ts` splits the response:
   - Text → `terminal:text-patch` → chat pane
   - JSON → `terminal:ir-deposit` → canvas pane
3. FlowDesigner subscribes to `terminal:ir-deposit` on its paneId (`canvas-editor`) and calls `convertIRToReactFlow()` to create nodes/edges

---

## Likely Failure Points

1. **Bus routing:** The terminal config has `links.to_ir: canvas-editor` — verify the bus.send targets the right paneId
2. **Subscription mismatch:** FlowDesigner subscribes on its own paneId. If the message targets a different paneId, it won't be received
3. **terminalResponseRouter:** May not be detecting JSON blocks correctly, or not sending `terminal:ir-deposit` at all
4. **Message target:** The `terminal:ir-deposit` message may have target `canvas-editor` but FlowDesigner line 532 checks `msg.target !== star && msg.target !== paneId` — if paneId doesn't match, it's dropped
