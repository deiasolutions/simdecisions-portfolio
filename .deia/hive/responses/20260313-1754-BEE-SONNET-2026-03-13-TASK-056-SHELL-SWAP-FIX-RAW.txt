## Summary

Fixed shell pane swap to preserve component state (terminal history, editor content, scroll position) by swapping ONLY app content fields while keeping node IDs stable for React key preservation.

## Key Fix

The old implementation swapped ALL fields including `id`, causing React to remount components and lose state. The new implementation:
- Swaps ONLY content: `appType`, `appConfig`, `label`, `audioMuted`, `busMute`, `notification`, `appState`
- NEVER swaps: `id`, `type`, `children`, `ratio`, `direction`, `loadState`, etc.
- Rejects swapping non-app nodes (splits/tabs)
