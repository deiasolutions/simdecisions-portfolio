## What Was Fixed

### File Modified
- `pyproject.toml` — Added `norecursedirs` configuration

### Change Details
```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
python_files = ["test_*.py"]
python_functions = ["test_*"]
asyncio_mode = "auto"
norecursedirs = ["_tools", ".git", ".deia", "node_modules", "build", "dist"]  # ← Added
```

### Why This Worked
1. **Problem:** `_tools/tests/test_cli_status_validation.py` and `test_migrate_statuses.py` were being collected by pytest even though they're not part of the main test suite
2. **Symptom:** These files have import statements (`from inventory_db import ...`) that only work when run from `_tools/` directory
3. **Impact:** When pytest collected them, it raised `ModuleNotFoundError`, blocking the entire test suite
4. **Solution:** Explicitly exclude `_tools` and other non-test directories from pytest collection
