## Root Cause Analysis
The PHASE-IR port placed v2.0 metadata (generators, resource_pools) inside the `flow.metadata` field, but the v2.0 detection and loader code only checked for these keys at the top level of the flow dict. This caused generators to not be registered, leading to simulations stopping early with minimal token creation.

## What Was Learned
When porting IR schemas, ensure all detection and loading logic checks both canonical locations (top-level) and common nesting patterns (metadata field). The Flow dataclass uses `metadata` as an extension point, so v2.0 loaders must check both places.
