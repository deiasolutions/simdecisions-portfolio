## Key Findings

### Why Cross-Window Leaks Are Impossible

1. **Per-Window Instances**: Each browser tab creates its own Shell → its own MessageBus instance
2. **Private _subs Map**: Each MessageBus has instance-local subscriber map (no sharing)
3. **No Sync Mechanism**: No BroadcastChannel, localStorage events, or service worker relay
4. **JavaScript Isolation**: Object references prevent cross-window access

### Root Cause

If users observe message cross-contamination, the actual causes are:
- **Same-window behavior** (both EGGs in one tab — expected behavior)
- **App subscriber logic issue** (wrong message type subscriptions)
- **Browser extension interference** (DevTools showing messages not delivered)
