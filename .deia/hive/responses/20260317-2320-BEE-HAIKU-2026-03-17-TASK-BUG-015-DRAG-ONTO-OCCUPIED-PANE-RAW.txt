## Summary

I have successfully fixed **TASK-BUG-015: Fix Drag Onto Occupied Pane in Stage**.

### The Problem
When users dragged a pane onto an occupied pane in Stage, there was no visual feedback (drop zone indicator or outline), making it appear as if occupied panes don't accept drops. The drop would actually work (MOVE_APP dispatch was correct), but users had no visual indication.

### The Root Cause
In `ShellNodeRenderer.tsx`, the `onDragOver` handler was setting `canAccept = true` for internal node moves (`hhs/node-id`), BUT it was not setting `isDragActive = true`. The visual outline is only shown when BOTH `canAccept && isDragActive` are true, so occupied panes showed no feedback.

### The Fix
**One-line fix** in `ShellNodeRenderer.tsx` line 160:
```tsx
if (e.dataTransfer.types.includes('hhs/node-id')) {
  setCanAccept(true);
  setIsDragActive(true);  // <- NEW: Enables outline feedback
}
```

This simple change ensures that when dragging a pane over any occupied pane:
1. ✅ The outline appears (visual feedback)
2. ✅ The DropZone indicator shows the zone type (center/left/right/top/bottom)
3. ✅ The drop works correctly, creating tabs (center) or splits (edges)
4. ✅ No regression to empty pane dragging
