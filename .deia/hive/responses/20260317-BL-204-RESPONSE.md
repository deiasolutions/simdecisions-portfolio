## Technical Details
The original overflow fallback logic (lines 88-103) had a fundamental misunderstanding:
- **Before:** When overflow detected, it tried to shift the position within current direction using Math.max/Math.min
- **After:** When overflow detected, it flips the direction flag and recalculates position anchor

Example (left edge):
- **Before:** `if (!openRight && x - menuWidth < 0) { x = Math.max(menuWidth, btnRect.right); }` — tries to shift x right but keeps `openRight=false`, causing menu to still render off-screen with `translateX(-100%)`
- **After:** `if (!openRight && x - menuWidth < 0) { openRight = true; x = btnRect.left; }` — flips direction to open right, anchors at button.left, no transform

This ensures the menu is always positioned and transformed correctly based on the final `openRight` and `openDown` flags.
