## Root Cause Analysis
The bug occurred when a malformed JWT token was passed via `?token=` query parameter. The original code would:
1. Attempt to parse the JWT
2. Fail silently (catch block with no action)
3. Clean the token from the URL anyway
4. Result: User sees LandingPage with no token and no error message

This created a confusing UX where users would be redirected after OAuth but see the landing page without any indication of what went wrong.

## The Fix
Modified `extractTokenFromUrl()` to:
1. Track whether token was successfully saved (`tokenSaved` flag)
2. Only clean URL via `window.history.replaceState()` if `tokenSaved === true`
3. Return early if token parsing fails (leave token in URL for debugging)
4. Add console.error() logging for malformed tokens to aid debugging
