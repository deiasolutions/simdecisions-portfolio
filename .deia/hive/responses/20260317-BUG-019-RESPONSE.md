### Root Cause
Canvas drag-drop handlers (`onDragOver`, `onDrop`) in `CanvasApp.tsx` were calling `preventDefault()` but NOT `stopPropagation()`, allowing drag events to bubble up to the Shell's pane drag system (`ShellNodeRenderer.tsx`), which intercepted them as pane swap operations.

### Fix Applied (Prior to This Task)
- Added `event.stopPropagation()` to `onDragOver` handler (line 418)
- Added `event.stopPropagation()` to `onDrop` handler (line 424)
- Added inline comments: `// Prevent shell pane drag system from intercepting`
