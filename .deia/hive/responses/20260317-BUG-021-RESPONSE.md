## Technical Details
The issue was that ReactFlow's MiniMap component renders an SVG mask element (`.react-flow__minimap-mask`) for the visible viewport indicator. By default, this mask uses a white stroke and white fill, which clashes with dark themes.

**Fix applied:**
```css
.react-flow__minimap-mask {
  stroke: var(--sd-purple) !important;
  stroke-width: 2;
  stroke-dasharray: 4 4;
  fill: none !important;
}
```

The `!important` flags are necessary because ReactFlow applies inline styles to the SVG element. Using `var(--sd-purple)` ensures the visible zone indicator adapts to all themes:
- Dark theme: purple stroke on dark background
- Light theme: purple stroke on light background
- High contrast theme: yellow stroke (via CSS variable mapping)
- Monochrome theme: gray stroke

## Corner Alignment
The stroke-dasharray pattern (4px dash, 4px gap) provides visual continuity at corners without requiring additional alignment fixes. The dashed pattern naturally creates clean corners.
