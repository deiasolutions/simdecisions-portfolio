## Technical Notes

### Icon Rendering Strategy
The `isTextIcon()` function uses a heuristic to distinguish icon types:
1. Length ≤ 2 chars → Unicode/emoji
2. Emoji Unicode ranges (U+1F300-1F9FF) → Unicode/emoji
3. Box drawing/geometric shapes (U+2500-27BF) → Unicode/emoji
4. Otherwise → CSS class

This allows TreeBrowser to support both:
- Modern emoji-based icons (palette, channels, etc.)
- Legacy CSS class icons (filesystem, if needed)

### Drag-and-Drop Architecture
1. **TreeNodeRow** → sets `dataTransfer` with MIME type from `meta.dragMimeType`
2. **PaletteAdapter** → provides `application/sd-node-type` MIME + `{ nodeType }` data
3. **CanvasApp** → reads MIME type, calls `reactFlow.screenToFlowPosition()`, creates node
4. **canvasInternal flag** → prevents Shell's pane drag system from intercepting canvas drags

### CSS Variables Compliance
All modified code uses `var(--sd-*)` CSS variables only. No hardcoded colors introduced.
