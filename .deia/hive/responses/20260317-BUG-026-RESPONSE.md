## What Was Done

### Root Cause
The bug occurred when the API returned malformed data where `data.items` was not an array (e.g., an object, null, or string). The original code used `data.items || []` which would NOT fall back to an empty array if `data.items` existed but was a non-array truthy value (like `{}`).

### Fixes Applied

1. **useKanban.ts (lines 47-58):**
   - Replaced simple ternary with explicit array validation
   - Now checks: `if (Array.isArray(data))` → use data directly
   - Then checks: `else if (data && Array.isArray(data.items))` → use data.items
   - Falls back to `[]` for all other cases (null, object, string, undefined)

2. **useKanban.ts (lines 90-100):**
   - Applied same fix to `fetchColumns()` for consistency
   - Ensures `columns` is always an array

3. **KanbanPane.tsx (lines 129-131):**
   - Added defensive guard: `const safeItems = Array.isArray(items) ? items : [];`
   - All references to `items` changed to `safeItems` (3 locations: filter, moveCard, count display, loading check)
   - Prevents crash even if useKanban somehow returns non-array
