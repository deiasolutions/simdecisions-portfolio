The bug report states "clicking channels does nothing" but code inspection reveals:

**1. Channel Click → Bus Event (WORKING)**
- `treeBrowserAdapter.tsx:173-187` - When `adapter === 'channels'` and user clicks a channel node:
  - Extracts `channelId`, `channelName`, `type` from `node.meta`
  - Emits `channel:selected` bus event with `target: '*'` (broadcast)

**2. Bus Event → Text-Pane Loads Messages (WORKING)**
- `SDEditor.tsx:356-386` - When in `chat` mode and `channel:selected` fires:
  - Sets label to `#${channelName}`
  - Resets chat state (timestamps, message count)
  - Fetches messages via `GET /efemera/channels/{channelId}/messages`
  - Formats messages as `**Author:** content` and renders as chat bubbles
  - Falls back to welcome message on API error

**3. Bus Event → Terminal Tracks Channel (WORKING)**
- `useTerminal.ts:176-186` - When `routeTarget === 'relay'` and `channel:selected` fires:
  - Updates `activeChannelId` and `activeChannelName` state
  - Enables relay mode message sending to the selected channel

### Architecture Verification
**MessageBus routing pattern:**
- Each pane subscribes via `bus.subscribe(nodeId, handler)`
- Channels adapter emits with `target: '*'` (broadcast)
- All subscribers receive event simultaneously
- Handlers filter by message type (`channel:selected`)
- Multiple panes can react to same event (text-pane + terminal)

**Channel data flow:**
1. Backend: `hivenode/efemera/store.py` - SQLite DB with channels table
2. API: `GET /efemera/channels` returns channel list with metadata
3. Adapter: `channelsAdapter.ts` groups into Pinned/Channels/DMs
4. Tree-browser: Renders grouped tree with unread badges
5. User clicks → Bus event → Subscribers load/update
