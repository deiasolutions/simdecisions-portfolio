## What Was Done

**Root Cause:**
The `treeBrowserAdapter.tsx` was sending `file:selected` bus events without a `name` field, and the `uri` field was missing the volume protocol prefix (e.g., `home://`). The `SDEditor.tsx` component expected `message.data.name` to display loading text and extract the filename for the label.

**Fix Applied:**
1. Modified `treeBrowserAdapter.tsx` line 189-206 to:
   - Extract protocol from config (default: `home://`)
   - Construct full URI with protocol prefix: `uri = ${protocol}${path}`
   - Add `name: node.label` to the event data
   - Keep backward-compatible `path` field

2. The SDEditor already handles the event correctly — it was just receiving incomplete data.

## Technical Details

**Before (broken):**
```typescript
bus.send({
  type: 'file:selected',
  data: {
    uri: 'README.md',  // ❌ Missing protocol
    path: 'README.md',
    // ❌ Missing name field
    size: 1024
  }
})
```

**After (fixed):**
```typescript
const protocol = (paneConfig as any).protocol || 'home://'
const path = node.meta.path as string
const uri = `${protocol}${path}`

bus.send({
  type: 'file:selected',
  data: {
    uri,              // ✅ home://README.md
    path,             // ✅ README.md
    name: node.label, // ✅ README.md
    size: 1024
  }
})
```

The backend `/storage/read` endpoint expects URIs in the format `protocol://path` (e.g., `home://README.md`), and the SDEditor uses the `name` field to display loading text and set the pane label.
