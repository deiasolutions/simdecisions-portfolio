### Root Cause:
The fix spec generators were called **before** the original spec was moved from `_active/` to `_done/`. They used `original_spec.path` directly, which pointed to the `_active/` location. By the time the fix spec was processed, the original had been moved to `_done/`, making the reference invalid.

### Solution:
The fix computes the `_done/` path at generation time:
- `original_spec.path.parent.parent / "_done" / original_spec.path.name`
- This works because the path goes up two levels from `_active/`, then down into `_done/`
