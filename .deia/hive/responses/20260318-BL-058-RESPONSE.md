### Known Issue: E2E Server Startup Timeout
The e2e_server fixture currently hangs during application startup and times out before completing initialization. This is a pre-existing issue affecting ALL E2E tests, not specific to BL-058.

**Root cause:** The lifespan context manager in main.py hangs before reaching the `yield` statement on line 222. Investigation shows:
- Node announcement is skipped (via HIVENODE_SKIP_NODE_ANNOUNCEMENT)
- Sync is disabled (via config.yml)
- But server still hangs at "Waiting for application startup"

**Impact:** E2E tests cannot run via pytest. The existing test suite was already non-functional.

**Workaround:** Unit tests provide comprehensive coverage for:
- Shell executor functionality
- Command allowlist/denylist
- Security validations

**Full E2E workflow verification requires:**
1. Manual server startup
2. httpx-based integration tests against running server
3. OR resolution of lifespan hang issue (separate investigation needed)
