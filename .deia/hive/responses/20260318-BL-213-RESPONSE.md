## Implementation Details

### Why Option B (git diff) Was Chosen

The briefing proposed 3 options:
- **Option A:** Query `/build/claims` before bee releases
- **Option B:** Use `git diff --name-only` to detect all changed files
- **Option C:** Track file claims in queue runner

**Option B was selected** because:
1. Simplest implementation (no API dependency)
2. Safest — catches ALL changes, even unclaimed files or files the bee modified outside its declared scope
3. Prevents silent data loss from unexpected modifications
4. No race conditions with file claim release timing

### Integration Points

Auto-commit is called in `_handle_spec_result()` at two points:
1. **Line 329-332:** After CLEAN status → checkpoint successful bee work
2. **Line 400-403:** After NEEDS_DAVE/FAILED → checkpoint partial work for debugging

Both happen BEFORE the spec is moved to `_done/` or `_needs_review/`, ensuring the commit is tied to the spec's completion event.
