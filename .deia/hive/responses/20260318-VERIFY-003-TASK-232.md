## Notes on CSS Shadow Variable

The response document noted that line 277 has `box-shadow: 0 -4px 12px var(--sd-shadow-lg);` which appears problematic:
- `--sd-shadow-lg` is a complete shadow definition: `0 8px 24px rgba(0, 0, 0, 0.3)`
- Prepending offset values `0 -4px 12px` before the variable is syntactically confusing

**Current state:** The line reads `box-shadow: 0 -4px 12px var(--sd-shadow-lg);` which is what the response flagged as needing correction to `box-shadow: var(--sd-shadow-lg);`

**Impact:** The syntax is valid CSS but semantically awkward. The variable will override the prepended values (CSS cascade rules). This could be cleaned up in a follow-up task for clarity, but functionally it works.
