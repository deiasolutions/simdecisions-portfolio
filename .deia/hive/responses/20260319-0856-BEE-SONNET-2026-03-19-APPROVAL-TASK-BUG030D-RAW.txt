The bee added a defensive deduplication layer at lines 86-95 of `chatHistoryAdapter.ts`:

```typescript
// DEFENSIVE DEDUPLICATION: Even though chatApi should dedupe,
// ensure each conversation ID appears only once.
const deduped = new Map<string, Conversation>();
for (const conv of conversations) {
  const existing = deduped.get(conv.id);
  if (!existing || new Date(conv.updated_at) > new Date(existing.updated_at)) {
    deduped.set(conv.id, conv);
  }
}
```

This ensures that even if `chatApi.listConversations()` returns duplicates (from API layer failures or volume merging issues), the adapter will guarantee each conversation appears exactly once in the tree-browser.
