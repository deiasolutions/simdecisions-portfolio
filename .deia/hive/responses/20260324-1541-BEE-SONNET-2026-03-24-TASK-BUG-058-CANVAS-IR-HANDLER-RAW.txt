## TASK-BUG-058 Complete

I've successfully fixed the canvas2 IR pipeline issue. Here's what was done:

### Root Cause
The canvas2 and canvas EGG prompts were instructing the LLM to return bare JSON arrays `[{...}]`, but `terminalResponseRouter.ts` expects envelope format `{"to_user": "...", "to_ir": [{...}]}`. When it saw responses starting with `[`, it treated them as plain text and used a fallback envelope with only `to_user` — no `to_ir` slot, so FlowDesigner never received the mutations.

### Fix Applied
1. **Updated `eggs/canvas2.egg.md`** prompt block (lines 268-287) to instruct LLM to use envelope format
2. **Updated `eggs/canvas.egg.md`** prompt block (lines 311-330) with the same fix
3. **Created regression test** `FlowDesigner.envelope-ir-deposit.test.tsx` with 7 test cases

### New Prompt Format
```json
{
  "to_user": "Brief confirmation message (1-2 sentences)",
  "to_ir": [
    { "type": "insert", "path": "/diagram/nodes/0", "value": {...} }
  ]
}
```
