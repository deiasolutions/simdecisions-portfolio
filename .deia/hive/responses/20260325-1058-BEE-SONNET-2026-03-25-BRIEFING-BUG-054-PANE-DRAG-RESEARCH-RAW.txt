### Key Findings

**Root Cause:** Empty panes have `loadState: 'COLD'` by default. The COLD state wrapper in `ShellNodeRenderer.tsx` (lines 234-249) **does not attach drag event handlers** (`onDragOver`, `onDrop`, `onDragLeave`). Without these handlers, the browser never calls `preventDefault()` and rejects the drop by default.

**Why tests pass but the UI doesn't work:** The test in `moveAppOntoOccupied.test.ts:274-353` calls the reducer directly, bypassing the rendering layer. The reducer logic is correct — the rendering layer blocks the interaction.

**The Fix (3 lines):**

In `ShellNodeRenderer.tsx:236`, add these three props to the COLD wrapper div:
```tsx
onDragOver={isDropTarget ? onDragOver : undefined}
onDrop={isDropTarget ? onDrop : undefined}
onDragLeave={isDropTarget ? onDragLeave : undefined}
```

And add drop zone rendering (line 318 pattern):
```tsx
{dropZone && <DropZone zone={dropZone} canAccept={canAccept} />}
```

This mirrors the HOT state implementation (lines 308-319) which already works correctly.
