## What Was Done

### Core Component (SCKeyboard.tsx)
- Single component with dual-layout rendering: portrait renders 4-row compact layout, landscape renders full 104-key layout
- Orientation detection via `screen.orientation` API with `window.innerWidth/innerHeight` fallback
- Automatic layout switching on orientation change
- All state management inline: buffer, shifted, capsActive, symbols, returnLocked, fnHeld, fnLocked, minimized, accentIdx, brightness, soundEnabled, readoutPos, bufferWindow, flowMode, fRowVisible, metaHeld, eightOSHeld
- Zero surface controls — only keys + buffer bar visible

### Compact Layout (Portrait)
- Symbol toggle (`#+=`) swaps alpha ↔ symbol rows
- Shift capitalizes next letter then auto-releases
- Backspace, space, return keys
- Status line with flow/hold/mute/caps indicators

### Full Layout (Landscape)
- F-row: Esc + F1-F12 (with Fn actions)
- Dual-symbol keys: number row, punctuation rows
- Shift/CapsLock/Meta/Fn/8OS modifiers
- Fn+F1-F7 dispatch actions (brightness, accent cycle, sound toggle, position cycle, flow/hold toggle, command palette)
- Fn+Arrows map to Home/End/PgUp/PgDn (per spec section 7)
- Return lock: double-tap toggles CR/send modes
- 8OS tap = command palette, 8OS hold+key = meta_8os modifier (per spec section 7)
- Escape triggers Escape Protocol (stub, wired in Wave B)

### Sub-Components
- **BacklitKey.tsx**: Key renderer with backlit/glass modes, cap color `#0e0f13`, flare animation
- **TrayGlowLayer.tsx**: SVG overlay with Gaussian blur for per-key LED glow, tray color `#060810`
- **ComposeBuffer.tsx**: Buffer bar with cursor, sliding window, hidden count, flow/hold indicator, floating submit bubble

### Hooks
- **useFnLayer.ts**: Fn single-tap toggles held, double-tap toggles locked. Dispatches Fn actions and Fn+Arrow navigation.
- **useFlare.ts**: 500ms linear decay flare animation via requestAnimationFrame
- **useClickSound.ts**: Cherry MX Blue synthesis (two-oscillator double-bump via Web Audio API)

### Registration
- **keyboardAdapter.tsx**: Maps AppRendererProps to SCKeyboard props, extracts ShellCtx bus (null-safe, wired in Wave B)
- **apps/index.ts**: Registered `registerApp('sc-keyboard', KeyboardAdapter)`
- **shell/constants.ts**: Added `{ appType: 'sc-keyboard', label: 'Keyboard', category: 'primitive' }` to APP_REGISTRY

### Tests

  - Orientation detection and layout switching

  - Full layout behaviors (F-keys, Fn actions, CapsLock, Escape, Return lock, 8OS)

  - Visual elements (status line, backlit keys, no native inputs)

### Stubs for Later Waves

### Key Implementation Details
- No native `<input>` or `<textarea>` elements — buffer is pure state + span rendering
- All CSS uses `var(--sd-*)` variables except constant backlight colors (`CAP_COLOR`, `TRAY_COLOR`)
- Dual-symbol keys extracted primary/secondary from label strings (e.g. `'1!'` → `'1'` or `'!'` based on shift state)
- Unique React keys for duplicate labels (Shift, 8OS, Meta, Alt) via `${key}-${idx}` pattern
- Cursor blink animation via 530ms interval (matching terminal primitive)
- Flow mode auto-commits at buffer window boundary (stub — Wave B implements)
