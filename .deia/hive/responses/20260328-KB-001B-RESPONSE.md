## What Was Done

### 1. Bus Emission Hook (`useKeyboardBus.ts`)
- Created hook exporting 5 emitters: `emitKeystroke`, `emitBufferSubmit`, `emitRTD`, `emitCommandPalette`, `emitEscapeProtocol`
- Keystroke events include key + modifiers (shift, ctrl, alt, fn, meta_8os) + timestamp
- Buffer submit sends text payload
- RTD events debounced to 300ms window to prevent spam
- All emitters are null-safe (no-op when bus unavailable)

### 2. Theme Subscription Hook (`useKeyboardTheme.ts`)
- Subscribes to ShiftCenter theme system via localStorage (`sd:hhpanes_theme`, `sd:hhpanes_mode`)
- Returns `KeyboardTheme` object: `dark`, `accentRgb`, `highContrast`, `fontMono`, `fontSans`
- Listens for storage events (cross-tab sync) and `theme:changed` bus messages (same-tab)
- Derives accent color from theme preview registry (THEME_REGISTRY from ThemeMenu.tsx)
- User accent override (Fn+F3) takes precedence over theme accent
- High contrast mode detection (`theme === 'high-contrast'`)
- Fallback to default purple accent if theme not found

## Notes

- Theme system uses THEME_REGISTRY from ThemeMenu.tsx as source of truth
- User accent override (Fn+F3) stored in local state, takes precedence over theme accent
- RTD debouncing prevents event spam (300ms window)
- High contrast mode (`effectiveBrightness = 1.0`) forces maximum brightness regardless of Fn+F1/F2
- Glass rendering path already exists in BacklitKey from Wave A — theme system just controls the dark flag
- Keyboard adapter already passes bus from ShellCtx (no changes needed)
- APP_REGISTRY bus patterns support wildcard matching for RTD events (`keyboard:*`)

## Architecture

```
SCKeyboard
  ├─ useKeyboardBus (emit keystrokes, RTD, commands)
  │    ├─ emitKeystroke(key, modifiers)
  │    ├─ emitBufferSubmit(text)
  │    ├─ emitRTD(event, payload)
  │    ├─ emitCommandPalette()
  │    └─ emitEscapeProtocol()
  │
  ├─ useKeyboardTheme (subscribe to theme system)
  │    ├─ dark (from sd:hhpanes_mode)
  │    ├─ accentRgb (from theme or user override)
  │    ├─ highContrast (from theme id)
  │    ├─ fontMono (from CSS vars)
  │    └─ fontSans (from CSS vars)
  │
  └─ useFnLayer (Fn state + RTD)
       ├─ fnHeld
       ├─ fnLocked
       └─ emitRTD('keyboard:fn-locked')
```
