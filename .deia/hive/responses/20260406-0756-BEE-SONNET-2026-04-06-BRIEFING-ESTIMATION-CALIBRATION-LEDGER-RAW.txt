## Summary

I've designed a comprehensive **estimation calibration ledger** system that tracks Clock, Cost, and Carbon for every dispatched task, computes calibration factors by task type, and provides budget projections. The design includes:

### **Architecture:**
- **Two new tables**: `inv_estimates` (per-task records) and `inv_calibration` (per-type factors)
- **Data flow**: Scheduler TASKS → estimates → build monitor/response files → actuals → calibration → projections
- **SQLAlchemy Core pattern** (matches existing inventory tables)
- **Works on both SQLite and PostgreSQL** (Railway)

### **Data Collection:**
1. **Estimates** from `scheduler_mobile_workdesk.py` TASKS list (hours directly, cost/carbon derived)
2. **Actuals** from build monitor API + response file Clock/Cost/Carbon sections
3. **Token rates** from `model_rates.yml`
4. **Carbon factors** from `carbon.yml` (energy × carbon intensity)

### **Calibration Algorithm:**
- Simple rolling average: `factor = mean(actual / estimate)` per task type
- Groups: spec, build, test, verify, css
- Auto-updates after each completed task
- Applied to new estimates: `calibrated = original × factor`

### **CLI Interface (`_tools/estimates.py`):**
- `import-scheduler` - bulk import from TASKS list
- `import-actuals` - pull from build monitor API
- `import-responses` - bulk import from response files
- `calibration` - show per-type factors
- `compare` - per-task original → calibrated → actual
- `budget --remaining` - project remaining work with calibrated estimates
- `trend` - accuracy improvement over time
