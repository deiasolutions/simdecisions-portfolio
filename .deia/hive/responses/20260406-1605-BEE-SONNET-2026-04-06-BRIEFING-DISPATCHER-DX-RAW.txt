### Summary

**Root Cause:** The scheduler's `extract_task_id()` function (lines 114-120 in `scheduler_daemon.py`) **collapses task IDs with letter suffixes**.

**The Bug:**
When extracting task IDs from filenames like `SPEC-WAVE0-A-ddd-directories.md`, the function:
1. Strips "SPEC-" → `WAVE0-A-ddd-directories`
2. Splits on dashes → `['WAVE0', 'A', 'ddd', 'directories']`
3. Finds first part with digit → `'WAVE0'` (contains '0')
4. Returns only up to that part → **"WAVE0"**

**Result:** All four specs (WAVE0-A, WAVE0-B, WAVE0-C, WAVE0-D) collapse to the same task_id **"WAVE0"**.

**Evidence:**
- `schedule.json` shows four tasks all with `task_id: "WAVE0"` and `status: "unknown"`
- All four distinct spec files still exist in `backlog/`
- Dispatcher shows `dispatched: 0, skipped: 0` because scheduler never marks them as "ready"

**Recommended Fix:** Modify `extract_task_id()` to include single-letter suffixes when they follow a numeric part. Add logic to check if the next part after a digit is a single uppercase letter (A-Z) and include it in the extracted ID.

**Blast Radius:** Low — the fix is backward-compatible with existing specs like `MW-031`, `EFEMERA-CONN-05`, etc.
