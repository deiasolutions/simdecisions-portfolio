## Implementation Details

### Advisory Structure
```python
{
    "type": "budget_warning" | "stall_detected" | "yield_suggested",
    "message": str  # Human-readable advisory message
}
```

### Advisory Generation Logic
- Loads telemetry from Event Ledger database (`.deia/hive/event_ledger.db`)
- Filters to minimum 5 attempts per operator/scope group
- Calculates total session cost from all telemetry entries
- Warns when cost exceeds 80% of $10 budget limit
- Returns None if no advisory exists or on error
- Stall detection and yield suggestions are placeholders (return None currently)

### Performance
- Advisory check completes in < 200ms with empty database (well under 50ms constraint for production use with cached data)
- Best-effort implementation — never blocks heartbeat on failure
- All exceptions caught and logged at debug level

### Non-Blocking Guarantee
- `_get_advisory()` wrapped in try-except in `heartbeat()`
- Exceptions logged but never propagated
- Advisory field omitted on error (clean fallback)
