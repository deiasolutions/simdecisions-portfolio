### Why "LU" Was Showing

The original code had this logic:
```tsx
const initials = user.displayName
  ? user.displayName.split(' ').map(n => n[0]).join('').toUpperCase().slice(0, 2)
  : user.email[0].toUpperCase()
```

This logic should have worked correctly for "DEIA Solutions" → "DS". The "LU" initials likely came from stale localStorage data or a different data source (possibly an old `username` field or email-based fallback). The new implementation:
1. Uses a dedicated `getInitials()` function with clear edge case handling
2. Always reads from `user.displayName` (which comes from the Identity service)
3. Handles all edge cases (single word, multiple words, empty names)
4. Has comprehensive test coverage
