## Patterns Fixed

**Root cause:** `replaceNode` returns `BranchesRoot | ShellTreeNode`, but `withUndo` expects `BranchesRoot`. Spread objects lose their specific type and need explicit assertion.

**Solution pattern:**
```typescript
// Before:
replaceNode(state.root, nodeId, { ...node, prop: value })

// After:
replaceNode(state.root, nodeId, { ...node, prop: value } as ShellTreeNode)
```

**For withUndo calls:**
```typescript
// Before:
withUndo(state, replaceNode(...), label)

// After:
withUndo(state, replaceNode(...) as BranchesRoot, label)
```
