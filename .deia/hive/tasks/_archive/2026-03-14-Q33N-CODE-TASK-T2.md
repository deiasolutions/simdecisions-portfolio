## Specification

Build the card grid home page as a React component. This is the visual surface for `apps.shiftcenter.com` — a chiclet/card grid showing all available EGGs.

### 1. AppsHome.tsx — main component

1.1. Accepts `eggs: EggMeta[]` prop (from registry service — see data shape below)
1.2. Renders search input at top (filters by displayName, description, egg ID — case-insensitive)
1.3. Groups eggs by `section` field into three sections with headers
1.4. Responsive grid: `repeat(auto-fill, minmax(200px, 1fr))` with 12px gap
1.5. Section headers: sentence case ("Core Products", "Productivity", "Platform Tools"), uppercase, 13px, `var(--sd-text-secondary)`, letter-spacing 0.05em
1.6. Empty state: "No apps match your search" when filter returns zero results
1.7. Accepts optional `bus` prop for emitting `egg:inflate` events

### 2. AppCard.tsx — individual chiclet

2.1. Icon: 40x40px rounded square (`border-radius: 8px`), background from `color` field mapped to `var(--sd-*)` variables, single letter (first char of displayName)
2.2. Name: 15px, weight 500, `var(--sd-text-primary)`
2.3. Description: 13px, `var(--sd-text-secondary)`, max 2 lines with `overflow: hidden; display: -webkit-box; -webkit-line-clamp: 2`
2.4. Footer row: status badge + optional subdomain link + version
2.5. Status badge colors — ALL via CSS variables:
  - BUILT: `var(--sd-green)` bg with `var(--sd-green-dimmer)` text
  - PARTIAL: `var(--sd-teal, var(--sd-green))` bg (fallback to green if teal not defined)
  - STUB: `var(--sd-orange)` bg with `var(--sd-orange-dimmer)` text
  - SPEC: `var(--sd-blue, var(--sd-purple))` bg (fallback to purple if blue not defined)
2.6. `onClick`: if `subdomain` exists, `window.location.href = 'https://' + subdomain`; else emit bus event `egg:inflate` with egg ID (or no-op if no bus)
2.7. Hover: border transitions from `var(--sd-border)` to `var(--sd-border-focus)`
2.8. Card background: `var(--sd-surface-alt)`
2.9. Card border: `1px solid var(--sd-border)`, border-radius `10px`
2.10. Card padding: `16px`

### 3. AppsHome.css — styles

3.1. ALL colors via `var(--sd-*)` CSS custom properties. NO hardcoded hex, rgb(), or named colors. Zero exceptions.
3.2. No external font imports. Use whatever the SC theme provides.
3.3. File must be under 100 lines.
3.4. Use `.apps-home-*` class prefix to avoid collisions.

### Data Shape

The component expects this TypeScript interface (defined in T3, but stub it locally if T3 isn't done yet):

```typescript
export interface EggMeta {
  egg: string;           // unique ID from frontmatter
  displayName: string;
  description: string;
  version: string;
  status: 'BUILT' | 'PARTIAL' | 'STUB' | 'SPEC';
  icon: string;          // single character (first char of displayName)
  color: string;         // CSS variable name, e.g. 'purple', 'green', 'orange'
  section: 'core' | 'productivity' | 'platform';
  subdomain: string | null;
}
```
