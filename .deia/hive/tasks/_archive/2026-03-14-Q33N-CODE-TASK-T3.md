## Specification

Build the service that provides the `EggMeta[]` array for the AppsHome component.

### 1. types.ts — shared types

Export the `EggMeta` interface:

```typescript
export interface EggMeta {
  egg: string;           // unique ID from frontmatter
  displayName: string;
  description: string;
  version: string;
  status: 'BUILT' | 'PARTIAL' | 'STUB' | 'SPEC';
  icon: string;          // single character (first char of displayName)
  color: string;         // color ramp name matching --sd-* vars (e.g. 'purple', 'green')
  section: 'core' | 'productivity' | 'platform';
  subdomain: string | null;
}

export type EggStatus = EggMeta['status'];
export type EggSection = EggMeta['section'];
```

### 2. parseEggFrontmatter.ts

2.1. `parseEggFrontmatter(rawMarkdown: string): Partial<EggMeta> | null`
2.2. Extracts YAML frontmatter between `---` delimiters
2.3. Parses the YAML manually (split by lines, key: value pairs — do NOT add a YAML parser dependency)
2.4. Returns partial EggMeta with fields from frontmatter: `egg`, `displayName`, `description`, `version`, `_stub`
2.5. Derives `icon` from first character of `displayName`
2.6. Handles missing optional fields gracefully:
  - `description` defaults to `""`
  - `version` defaults to `"0.0.0"`
  - `_stub` defaults to `false`
  - `displayName` defaults to `egg` value
2.7. Returns `null` for completely invalid input (no frontmatter delimiters)
