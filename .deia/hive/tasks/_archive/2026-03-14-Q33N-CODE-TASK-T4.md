## Specification

Wire the AppsHome component to the shell so it renders when the `apps-home` EGG is loaded.

### 1. Create `appsHomeAdapter.tsx`

Follow the same pattern as `browser/src/apps/canvasAdapter.tsx`:

```typescript
import { useContext } from 'react'
import { ShellCtx } from '../infrastructure/relay_bus'
import { AppsHome } from '../primitives/apps-home'
import { getRegistry } from '../services/egg-registry'
import type { AppRendererProps } from '../shell/components/appRegistry'

export function AppsHomeAdapter({ paneId, config }: AppRendererProps) {
  const ctx = useContext(ShellCtx)
  const bus = ctx?.bus || null
  const eggs = getRegistry()

  return <AppsHome eggs={eggs} bus={bus} />
}
```

### 2. Register in `browser/src/apps/index.ts`

Add import and registration:

```typescript
import { AppsHomeAdapter } from './appsHomeAdapter'
// ... in registerApps():
registerApp('apps-home', AppsHomeAdapter)
```

### 3. Add hostname mapping in `browser/src/eggs/eggResolver.ts`

In the `hostnameMap` fallback object inside `resolveEggFromHostname()`, add:

```typescript
'apps.shiftcenter.com': 'apps-home',
```

This means `apps.shiftcenter.com` resolves to the `apps-home` EGG. For localhost dev, use `?egg=apps-home` or navigate to `/apps-home`.

### 4. Add to APP_REGISTRY in `browser/src/shell/constants.ts`

Add entry to the `APP_REGISTRY` array:

```typescript
{ appType: 'apps-home', label: 'App Directory' },
```

This ensures the FAB button shows "App Directory" as a creatable applet.

### 5. Verify

After wiring:
- `localhost:5173/apps-home` should render the AppsHome component
- `localhost:5173?egg=apps-home` should also work
- The FAB menu should show "App Directory" in the Applets submenu
- No console errors
