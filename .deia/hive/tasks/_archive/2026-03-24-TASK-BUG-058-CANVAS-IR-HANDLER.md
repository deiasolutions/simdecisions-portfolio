# TASK-BUG-058: Fix canvas2 IR Pipeline — Envelope Format

## Objective

Fix the canvas2 EGG IR pipeline so that PHASE-IR mutations generated by the LLM in the terminal are successfully delivered to FlowDesigner and rendered as nodes on the canvas.

## Context

**The Problem:**
IR mutations from terminal (paneId `canvas-ir`, routeTarget `ir`) are NOT reaching FlowDesigner in canvas2 EGG.

**Root Cause (DIAGNOSED by Q33N 2026-03-24):**
The canvas2 EGG prompt block (line 268-287 in `eggs/canvas2.egg.md`) instructs the LLM to return bare JSON mutation arrays:
```json
[
  { "type": "insert", "path": "/diagram/nodes/0", "value": {...} }
]
```

But `terminalResponseRouter.ts` expects envelope format with named slots:
```json
{
  "to_user": "I've generated the requested diagram changes.",
  "to_ir": [
    { "type": "insert", "path": "/diagram/nodes/0", "value": {...} }
  ]
}
```

Without envelope format, the response is treated as plain text and routed to `to_user` only. The `to_ir` slot is never populated, so FlowDesigner never receives the mutations.

**Why This Happens:**
1. `terminalResponseRouter.ts` calls `parseEnvelope(raw)` (line 141)
2. If the response starts with `[` instead of `{`, it's treated as plain text (line 116-121)
3. Plain text goes to fallback envelope: `{ to_user: raw }` — no `to_ir` slot
4. FlowDesigner never receives `terminal:ir-deposit` event

**The canvas EGG (working)** also has this issue in its prompt, but it works because the LLM happens to wrap responses in code fences, which `extractIRBlocks()` detects and routes separately. This is fragile and not the intended design.

**Architecture Review (Q33N findings):**
- FlowDesigner DOES subscribe to `terminal:ir-deposit` (lines 526-577 in FlowDesigner.tsx) ✓
- Terminal DOES route envelope slots correctly (useTerminal.ts lines 696-700) ✓
- PaneId routing is correct: `to_ir: "canvas-editor"` matches FlowDesigner's paneId ✓
- **The ONLY issue is the prompt format** — LLM is told to return bare arrays, not envelopes
