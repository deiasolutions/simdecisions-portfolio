# Tribunal Episode Template

Use this template to scaffold a new episode directory.

---

## Directory Structure

```
production/episodes/tribunal/{NNN}-{slug}/
├── topic.md              # Debate motion and rules (REQUIRED)
├── context.md            # Evidence for panelists (REQUIRED)
├── personas.md           # Persona assignments (OPTIONAL)
├── format.yaml           # Episode-level overrides (OPTIONAL, usually empty)
├── transcript.md         # Full debate text (OUTPUT)
├── metrics.json          # API costs and timing (OUTPUT)
├── README.md             # Episode status notes (OUTPUT)
├── OBS-SETUP.md          # Recording guide (OUTPUT)
├── RENDER-RESULTS.md     # TTS render summary (OUTPUT)
└── render/
    ├── manifest.json     # Audio timing metadata (OUTPUT)
    └── wavs/             # Sentence WAV files (OUTPUT)
```

Files marked **REQUIRED** are inputs you create before rendering.
Files marked **OUTPUT** are generated by the production pipeline.

## File Templates

### topic.md

```markdown
# Episode {NNN}: {Title}

## Topic

"{One-sentence debate motion}"

## Framing

{Describe what evidence is available to panelists. What projects, documents, or artifacts can they cite? What's the context?}

## Rules

- {Rule 1: e.g., "Reference specific technical capabilities, not generic flattery"}
- {Rule 2: e.g., "In rebuttals, name a specific panelist and challenge a specific claim"}
- {Rule 3: e.g., "Make your case and leave it to the audience to decide"}

## Context Mode

{Describe how context is injected. Options:
- "Full context injection — each model receives [X] as background in every prompt"
- "Minimal context — topic only, no supporting materials"
- "Evidence on request — panelists can ask for specific documents"}
```

**Example:** See `production/episodes/tribunal/001-who-should-hire-dave/topic.md`

---

### context.md

```markdown
# {Episode Title} — Evidence for Debate

{Intro paragraph explaining what evidence panelists should cite.}

---

## Evidence Item 1: {Name}

**Type:** {e.g., "GitHub repo", "Product demo", "Research paper"}

{Description of what this item is and what it demonstrates.}

**What this demonstrates:** {Key takeaway for panelists to cite.}

---

## Evidence Item 2: {Name}

{Repeat for each piece of evidence.}

---

## Key Themes Across All Evidence

1. {Theme 1}
2. {Theme 2}
3. {Theme 3}
```

**Example:** See `production/episodes/tribunal/001-who-should-hire-dave/context.md`

---

### personas.md (Optional)

```markdown
# Episode {NNN} — Panelist Personas

## Mode: {Raw | Assigned}

{If Raw:}
All panelists debate without assigned personas. Each model argues naturally.

{If Assigned:}
Each panelist is assigned a persona to embody during the debate.

| Panelist | Persona | Description |
|----------|---------|-------------|
| Claude | {Persona Name} | {What this persona cares about, how they argue} |
| GPT | {Persona Name} | {What this persona cares about, how they argue} |
| Gemini | {Persona Name} | {What this persona cares about, how they argue} |
```

**Example:** See `production/episodes/tribunal/001-who-should-hire-dave/personas.md` (Raw mode)

---

### format.yaml (Optional)

```yaml
# Episode {NNN} format overrides
# Inherits all defaults from production/shows/tribunal/format.yaml
# Uncomment below to override specific values:

# moderator:
#   model: "claude-sonnet-4-20250514"
#   voice: "bf_emma"
#
# phases:
#   - name: intro
#     speaker: moderator
#   - name: opening_statement
#     speaker: panelist
#     rounds: 3
#   - name: rebuttal
#     speaker: panelist
#     rounds: 3
#   - name: closing_argument
#     speaker: panelist
#     rounds: 3
#   - name: summary
#     speaker: moderator
```

Most episodes inherit the default format. Only override if you need custom phases, different round counts, or a different moderator model/voice.

**Example:** See `production/episodes/tribunal/001-who-should-hire-dave/format.yaml` (no overrides)

## Quick Start — New Episode

1. **Create directory:**
   ```bash
   mkdir -p production/episodes/tribunal/{NNN}-{slug}
   cd production/episodes/tribunal/{NNN}-{slug}
   ```

2. **Write topic.md:**
   - Define the debate motion
   - Set framing and rules
   - Decide context mode

3. **Write context.md:**
   - List evidence items panelists can cite
   - Include specific details, not vague summaries
   - Highlight key themes

4. **Write personas.md (optional):**
   - Default to Raw mode (no assigned personas)
   - Assign personas only if the topic demands role-play

5. **Generate transcript:**
   - [UNDOCUMENTED — manual prompting currently]
   - Compile into `transcript.md` with `## [phase] Speaker` headers

6. **Render audio:**
   ```bash
   python -m production.engine --episode {NNN}-{slug} --render
   ```

7. **Record in OBS:**
   - Set up VB-Audio cables
   - Configure OBS sources
   - Run replay and record

8. **Export and publish:**
   - Remux mkv to mp4 if needed
   - Edit/mix in post-production
   - Upload to podcast platform

## Naming Conventions

### Episode Numbering

- Zero-padded 3 digits: `001`, `002`, `010`, `100`
- Sequential across all Tribunal episodes
- No gaps — retired episodes stay in numbering sequence

### Slug Format

- Kebab-case (lowercase + hyphens)
- 3-6 words, descriptive of the topic
- No special characters, no spaces

**Good slugs:**
- `who-should-hire-dave`
- `is-ai-sentient`
- `best-programming-language`

**Bad slugs:**
- `Dave_Eichler_Debate` (underscores, capitalization)
- `episode-about-ai-and-stuff` (vague, too long)
- `002` (just a number, not descriptive)

## Output File Reference

These are generated by the pipeline — do not create them manually.

### transcript.md

Generated from debate generation process. Format:

```markdown
# Transcript: {Topic}

## [intro] Moderator

{Intro text}

## [opening_statement] Claude

{Claude's opening}

## [opening_statement] GPT

{GPT's opening}

{...continue for all segments...}
```

### metrics.json

Generated during debate generation. Tracks API costs:

```json
{
  "panelists": {
    "claude": {
      "input_tokens": 54022,
      "output_tokens": 1072,
      "api_cost_usd": 0.178146,
      "wall_clock_seconds": 45.16,
      "turns": 3
    },
    "gpt": { ... },
    "gemini": { ... }
  },
  "totals": {
    "api_cost_usd": 0.200727,
    "wall_clock_seconds": 80.38
  }
}
```

### manifest.json

Generated by TTS render. Includes per-segment timing:

```json
{
  "version": 1,
  "episode": "{NNN}-{slug}",
  "total_duration": 1043.75,
  "segments": [
    {
      "index": 0,
      "phase": "intro",
      "speaker_id": "moderator",
      "wav_files": ["intro_moderator_000_000.wav", ...],
      "durations": [11.375, 3.875, ...],
      "total_duration": 55.05,
      "sentence_count": 11
    },
    ...
  ]
}
```

### WAV Files

Generated by TTS render. Naming convention:

```
{phase}_{speaker_id}_{segment_index:03d}_{sentence_index:03d}.wav
```

Examples:
- `intro_moderator_000_000.wav`
- `opening_statement_claude_001_015.wav`
- `closing_argument_gemini_009_012.wav`

## Checklist Before Rendering

- [ ] `topic.md` defines clear debate motion
- [ ] `context.md` provides specific evidence (not vague summaries)
- [ ] `personas.md` exists (even if just Raw mode)
- [ ] `format.yaml` exists (even if empty/no overrides)
- [ ] `transcript.md` follows `## [phase] Speaker` header format
- [ ] Voice assignments are unique (no duplicate voices)
- [ ] Episode directory name is `{NNN}-{slug}` format
- [ ] All file paths are absolute when referencing in commands
